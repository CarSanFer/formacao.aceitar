🎯 Fluxo de Trabalho Recomendado

▾

Sequência lógica para construir um plano de trabalhos de raiz:

  1. Configurar o Projeto — Criar projeto em branco, definir calendário (horário, feriados, férias) e data de início

  2. Introduzir a WBS — Copiar lista de tarefas do Excel, organizar hierarquia com "Avançar Tarefa", definir Tarefas Resumo

  3. Definir Durações — Preencher durações em dias úteis em todas as tarefas (não nas resumo)

  4. Criar Ligações — Encadear tarefas (CI por defeito), aplicar desfasamentos onde necessário

  5. Inserir Milestones — Consignação, aprovações, conclusão de fases — duração 0 dias

  6. Verificar Caminho Crítico — Ativar visualização de tarefas críticas (vermelho), analisar Margem Total

  7. Atribuir Recursos — Configurar Folha de Recursos (MO/EQ/Material), atribuir às tarefas

  8. Extrair Cronogramas — Plano de MO, Plano de EQ, Cronograma Financeiro

  9. Guardar Linha de Base (Baseline) — Fixar o plano aprovado antes do início da obra

  10. Monitorização periódica — Definir Data de Estado, preencher % concluída e datas reais, analisar desvios

2.1 · Planeamento

Criar Projeto Novo

Ao iniciar o MS Project, selecionar "Projeto em Branco" para começar de raiz, ou utilizar um modelo pré-configurado com lista de recursos e vistas já definidas.

Vista principal — Gráfico de Gantt

▾

Por defeito, o projeto abre na vista Gráfico de Gantt, composta por 4 áreas:

📏 Linha Cronológica

Barra superior com resumo temporal de todo o projeto. Pode ser ativada/desativada em Ver

📋 Lista de Tarefas

Tabela com designação, duração, início, fim e recursos de cada tarefa

📊 Gráfico de Gantt

Representação gráfica das barras de duração e sequencialidade das tarefas

ℹ️ Barra de Estado

Permite alternar entre agendamento Manual/Automático e mudar rapidamente de vista

2.2 · Planeamento

Calendário do Projeto

Configurar horário de trabalho, exceções (feriados, férias) e semanas de trabalho específicas.

Como aceder

▾
Projeto › Alterar Tempo Útil

Por defeito: Calendário Padrão com horário 08h–12h e 13h–17h, Segunda a Sexta.

⚠️ Exceções

Registar feriados, períodos de paragem, etc. Exemplo: Natal — 25/12 com opção "Descanso".

📅 Semanas de Trabalho

Para obras com trabalho ao Sábado: criar nova entrada em "Semanas de Trabalho", selecionar "Sábado" e definir horário específico (ex. 08h–12h / 13h–17h).

ℹ️ Opções relevantes
Hora de início predefinida: 08:00 · Hora de fim: 17:00 · Horas/dia: 8 · Horas/semana: 40 · Dias/mês: 20
2.3 · Planeamento

Data de Início do Projeto

Definir data de início

▾
Projeto › Informação do Projeto › Data de Início

Alterar a data de início para a data pretendida. O Project ajustará automaticamente todas as tarefas a partir dessa data (no modo de agendamento automático).

2.4 · Planeamento

Agendamento de Tarefas

Manual vs. Automático — o agendamento automático é o modo recomendado para tirar partido total do MS Project.

Manual vs. Automático

▾
ModoComportamentoQuando usar
Automático ⭐O Project calcula Início e Conclusão com base nas ligações de tarefasRecomendado — permite controlo dinâmico do plano
ManualO utilizador define datas manualmente; o Project não as alteraTarefas com datas fixas por imposição contratual
Como alterar

Opção 1:

Tarefa › Modo de Agenda

Opção 2: clicar na Barra de Estado no fundo do ecrã → selecionar modo para novas tarefas

Opção 3: alterar tarefa a tarefa na coluna "Modo de Tarefa"

2.5 · Planeamento

Tipo de Tarefa

Define o que permanece constante quando se altera a equação Trabalho = Duração × Unidades.

Os 3 tipos

▾
TipoO que fica fixoExemplo prático
Unidades Fixas (padrão)Número de pessoas atribuídasAo alterar a duração, o trabalho adapta-se mas as unidades mantêm-se
Trabalho FixoTotal de horas de trabalhoAdicionar mais pessoas reduz a duração (ex. jardim — mais jardineiros = menos tempo)
Duração FixaDuração em diasReunião de 1h — adicionar pessoas não reduz a duração da reunião
ℹ️ Como alterar
Inserir coluna "Tipo" na Lista de Tarefas e editar diretamente, ou aceder por duplo clique na tarefa → Separador Avançado.

Como inserir colunas

Botão direito no cabeçalho de uma coluna → "Inserir Coluna" → escrever o nome da coluna pretendida. As colunas inserem-se à esquerda da selecionada e podem ser arrastadas.

Escala Temporal

Botão direito nas datas do Gantt → "Escala Temporal". Configurar 2 camadas:

  • Camada Média: Meses (M1, M2, M3 … desde o início)
  • Camada Inferior: Semanas (S1, S2, S3 … desde o início)
Dicas adicionais
  • Para quantidades/unidades: usar colunas de Texto (Texto1, Texto2...)
  • Para cálculos: usar colunas de Número
  • Para duração em dias de calendário: usar coluna Duração2 com fórmula: DateValue([Conclusão Agendada])-DateValue([Início Agendado])+1
  • Configurar estilos de barra: botão direito na área do Gantt → "Estilos de Barra"
2.7 · Planeamento

Introdução de Tarefas

Método recomendado: Copy/Paste do Excel

▾

  1. Preparar a lista de tarefas no Excel (coluna com os nomes das tarefas)

  2. Selecionar e copiar as células no Excel (Ctrl+C)

  3. Clicar na primeira linha da coluna "Nome da Tarefa" no Project

  4. Colar (Ctrl+V) — as tarefas são importadas diretamente

✅ Vantagem
Evita erros de digitação e acelera significativamente a construção do plano, especialmente em obras com muitas tarefas.
2.8 · Planeamento

WBS — Work Breakdown Structure

Estrutura hierárquica que divide o projeto em blocos controláveis, do nível mais alto (capítulo) até ao nível de tarefa individual.

Hierarquia e estrutura

▾
NívelExemploTipo no Project
1Projeto GlobalTarefa Resumo Projeto (nível 0)
2Arquitectura / EstruturaTarefa Resumo (capítulo)
3Polivalente / Edifício CTarefa Resumo (sub-capítulo)
4DemoliçõesTarefa Resumo
5Demolição de paredes interioresTarefa (com duração real)
Como criar a hierarquia

Selecionar as tarefas que são sub-tarefas de uma Tarefa Resumo e clicar em:

Tarefa › Avançar Tarefa

Ou usar o atalho Alt+Shift+→

ℹ️ Regras
  • A duração da Tarefa Resumo é calculada automaticamente pelo Project
  • As sub-tarefas devem aparecer imediatamente após a Tarefa Resumo correspondente
  • Inserir coluna "WBS" para ver codificação automática
  • Inserir coluna "Nível Hierárquico" para visualizar o nível de cada tarefa
2.9 · Planeamento

Durações das Tarefas

Regras de preenchimento

▾
  • Preencher durações apenas nas Tarefas (não nas Tarefas Resumo)
  • A duração das Tarefas Resumo é calculada automaticamente pelo Project
  • Unidade padrão: dias úteis (configurável em Opções)
Duração em dias de calendário

Inserir coluna "Duração2" e configurar fórmula personalizada:

Fórmula
DateValue([Conclusão Agendada])-DateValue([Início Agendado])+1
2.10 · Planeamento

Marco de Projeto (Milestone)

Pontos ou eventos significativos do projeto com duração zero que servem para monitorizar datas-chave.

Exemplos e criação

▾

Exemplos típicos em obra:

Consignação
Assinatura de Contrato
Aprovação de Materiais
Aprovação de Subempreitadas
Conclusão de Fase
Receção Provisória
Data limite resposta DO
Como criar

Inserir tarefa com Duração = 0 dias → o Project converte automaticamente para Milestone (símbolo ◆ no Gantt).

⚠️ Gestão Contratual
Milestones de prazo de resposta do DO (BPEs/BAMEs) devem ser ligadas às tarefas dependentes, evidenciando o impacto no prazo da obra em caso de atraso.
2.11 · Planeamento

Ligações entre Tarefas

A principal funcionalidade do Project — as dependências direcionam a agenda: qualquer alteração à predecessora afeta automaticamente todas as sucessoras.

4 tipos de ligação

▾
CódigoTipoSignificadoUso típico
CIConclusão–Início ⭐B começa quando A terminaPadrão — sequência normal de trabalhos
CCConclusão–ConclusãoB termina quando A terminaTrabalhos que acabam em simultâneo
IIInício–InícioB começa quando A começaTrabalhos que arrancam em simultâneo
ICInício–ConclusãoB termina quando A começaRaro — situações muito específicas
Desfasamentos

Aplicar na coluna Predecessoras junto ao código de ligação:

  • Desfasamento negativo (sobreposição): ex. 44CI-2d → B começa 2 dias antes de A terminar
  • Desfasamento positivo (folga): ex. 44CI+5d → B começa 5 dias após A terminar
3 formas de criar ligações

  1. Via colunas: editar diretamente a coluna "Predecessoras" (ex: digitar "3" ou "3CI-2d")

  2. Via ribbon: selecionar tarefas →

    Tarefa › Ligar Tarefas Selecionadas
    ou Ctrl+F2

  3. Via painel de detalhes: ativar em Ver → "Detalhes" → editar antecessoras/sucessoras no painel inferior

2.12 · Planeamento

Sobreposições e Folgas

Desfasamentos positivos e negativos

▾
TipoDesfasamentoExemploEfeito

2.13 · Planeamento

Caminho Crítico

Sequência de tarefas com zero dias de folga — qualquer atraso origina derrapagem direta no prazo global da obra.

Ativar e analisar

▾
Formatar › Caminho Crítico

As tarefas críticas ficam a vermelho no Gantt.

Coluna Margem Total
  • Margem Total = 0 → Tarefa crítica
  • Margem Total > 0 → Número de dias que pode atrasar sem impactar o prazo global
  • Aplicar filtro na coluna para listar apenas as tarefas críticas
🔴 Atenção operacional
Monitorizar semanalmente o caminho crítico durante a execução da obra. Qualquer risco numa tarefa crítica deve ser escalado imediatamente.

Descrição

2.16 · Planeamento

Cronogramas de Mão-de-Obra e Equipamento

Gerar os cronogramas

▾

  1. Garantir que todos os recursos têm o campo Grupo preenchido (mo ou eq) na Folha de Recursos

  2. Ir a:

    Recurso › Planeador de Equipa › Utilização de Recursos

  3. Quando solicitado, inserir "mo" (Mão de Obra) ou "eq" (Equipamento)

  4. Desativar a opção "Trabalho" (botão direito na tabela) — ficam apenas as unidades

  5. Configurar Escala Temporal: Camada Média = Meses / Camada Inferior = Semanas

  6. Guardar vista personalizada para reutilização futura: "Guardar Vista"

✅ Vista personalizada
Criar vista com nome "Cronograma de MO [JF]" ou "Cronograma de EQ [JF]" e guardar no modelo para reutilização em futuros projetos.
2.17 · Planeamento

Cronograma Financeiro

Como configurar

▾

Pré-requisito: coluna "Custo" preenchida em todas as tarefas.

  1. Criar nova vista: Mais Vistas → Nova → Vista Única

  2. Definir: Ecrã = Utilização de Tarefas / Tabela = Custo / Filtro = Todas as Tarefas

  3. Exportar para Excel: copiar e colar a tabela no Excel

⚠️ Limitação
O Project distribui os valores de custo aritmeticamente pela duração das tarefas — pode não refletir a curva real de faturação em obra. Validar sempre com o responsável financeiro.

Alternativa: usar Relatórios Visuais → "Relatório Fluxo Monetário" (exporta para Tabela Dinâmica Excel).

Linha de Base (Baseline)

O ponto de referência fixo a partir do qual se medem todos os desvios durante a execução da obra.

O que guarda e quando usar

▾

Ao guardar a Linha de Base, o Project copia 5 campos para campos de baseline:

Início
Fim
Duração
Trabalho
Custo
Projeto › Definir Linha de Base
Regra fundamental
🔴 Quando guardar
A Linha de Base só deve ser guardada após aprovação do plano e imediatamente antes do início efetivo da obra. Nunca gravar a baseline com um plano em rascunho.
Para que serve
  • Comparar plano previsto vs. execução real em qualquer momento da obra
  • Calcular desvios de prazo (Tabela de Desvio)
  • Alimentar relatório "Planeado x Realizado"
  • Evidenciar responsabilidades em caso de litígio contratual

Data de Estado

Data de referência para calcular o progresso do projeto e exibir o estado real de execução.

Configuração

▾
Projeto › Data de Estado

Definir a data do balizamento (ex.: final do mês). O Project usa esta data para calcular o progresso esperado e comparar com o real.

ℹ️ Boas práticas
Atualizar a Data de Estado antes de cada balizamento. A linha de progresso no Gantt mostra o corte temporal de análise.

BPEs, BAMEs e TNPs — Tempos de Resposta

▾

  • Em caso de atraso do Cliente: notificar imediatamente e salvaguardar impacto no prazo e preço
